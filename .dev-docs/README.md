# .dev-docs 目录说明

> **⚠️ 二次开发专用目录**

---

## 目录定位

```
Penguin-Magic/
├── docs/              # 原项目文档（不要修改）
│   ├── README.md
│   ├── IMPLEMENTATION_GUIDE.md
│   └── ...
│
└── .dev-docs/         # 二次开发文档（你可以自由编辑）← 你在这里
    ├── INDEX.md       # 文档导航索引
    ├── README.md      # 本文件
    ├── DEVELOPMENT_GUIDE.md
    └── PROJECT_ANALYSIS.md
```

---

## 与原项目 docs/ 的区别

| 特性 | `.dev-docs/` (本目录) | `docs/` (原项目) |
|------|---------------------|------------------|
| **用途** | 二次开发指南 | 原项目记录 |
| **编辑权限** | ✅ 可自由编辑 | ⚠️ 建议不修改 |
| **内容类型** | 实用代码指南、分析 | 设计文档、历史 |
| **目标读者** | 二次开发者 | 原项目维护者 |
| **可迁移性** | ✅ 随项目迁移 | 固定在原项目 |

---

## 本目录包含

- **DEVELOPMENT_GUIDE.md** - 完整的二次开发地图
  - 项目架构详解
  - 节点系统使用
  - API接入方式
  - 添加新功能步骤

- **PROJECT_ANALYSIS.md** - 项目现状分析
  - 技术栈说明
  - 关键文件列表
  - 配置存储方式

- **INDEX.md** - 快速导航索引

---

## 使用建议

### ✅ 推荐做法
1. 在本目录添加你的分析和计划文档
2. 随项目一起迁移到其他工作环境
3. 作为快速上手的参考

### ⚠️ 注意事项
1. 不要修改 `../docs/` 中的原项目文档
2. 如需参考原项目设计，请阅读 `docs/` 目录
3. 本目录文档与原项目文档分开维护

---

## 快速开始

```bash
# 查看二次开发索引
cat .dev-docs/INDEX.md

# 阅读开发指南
cat .dev-docs/DEVELOPMENT_GUIDE.md

# 查看项目分析
cat .dev-docs/PROJECT_ANALYSIS.md
```
